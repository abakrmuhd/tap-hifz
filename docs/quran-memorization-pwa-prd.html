Tap Hifz Quran Memorization PWA PRD

A focused memorization tracker where the primary action is tapping ayah numbers to record repetition strength for individual ayahs and transition strength between consecutive ayahs.

Draft PRD PWA Memorization Strength Offline First

Problem

Quran memorization is not only about repeating isolated ayahs. A learner also needs to strengthen the movement from one ayah into the next. Most simple trackers count repetitions globally, which hides whether weakness is inside an ayah or at the transition point.

Core Insight

The ayah number becomes the input surface. A single tap strengthens the ayah. A double tap strengthens the link from the previous ayah to the current ayah.

Core Interaction

Tap ayah number Increment the individual repetition count for that ayah.
Double tap ayah number Increment the transition count from the previous ayah to the current ayah. This starts from ayah 2 because ayah 1 has no previous ayah in the current passage. A double tap only counts the transition; it does not increment the individual ayah repetition count.
The app tracks two related but separate strengths: individual ayah repetition strength and consecutive ayah transition strength.

Target Users

  • Students memorizing new Quran passages.
  • Students reviewing weak ayahs and weak transitions.
  • Teachers or parents who want a quick visual signal of practice progress.

Goals

  • Make repetition logging fast enough to use during recitation practice.
  • Show which ayahs are individually strong or weak.
  • Show which transitions between ayahs need more practice.
  • Work well as a mobile-first installable PWA.

Functional Requirements

Area Requirement Notes
Ayah display Display Quran ayahs with clearly tappable ayah numbers. The ayah number should be the main control, not a separate button.
Individual count Single tap on an ayah number increments that ayah's repetition count by 1. Count is stored per surah and ayah.
Transition count Double tap on ayah N increments only the transition count from ayah N-1 to ayah N. Disabled or ignored for ayah 1. It does not also increment ayah N's individual count.
Strength feedback Convert counts into visual strength levels. Exact thresholds need product decision.
Persistence Save progress locally so it works offline. Sync can be a later feature.
Correction Allow undo or decrement when the user taps accidentally. Important because double tap gestures can misfire.

Data Model Draft

  • ayahProgress[surahId][ayahNumber].repetitionCount
  • transitionProgress[surahId][fromAyah][toAyah].repetitionCount
  • sessions[].events for optional practice history and undo support.

Success Metrics

  • User can log repetitions without breaking recitation flow.
  • User can identify weakest ayahs and weakest transitions within seconds.
  • Progress survives offline use and app reloads.

Open Product Decisions

Scope of Quran text Start with full Quran, selected surahs, or user-selected memorization passages?
Strength thresholds What counts as weak, building, strong, and mastered?
Gesture behavior Double tap counts only the transition from the previous ayah to the current ayah.
Review mode Should the app suggest the next weak ayah or transition to practice?

Initial MVP

  • Mobile-first PWA shell.
  • Surah or passage view with ayah text and tappable ayah numbers.
  • Single tap individual repetition tracking.
  • Double tap transition repetition tracking.
  • Local persistence.
  • Simple visual strength indicators for ayahs and transitions.
  • Undo last action.
